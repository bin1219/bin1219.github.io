github使用

github命令

git上配置github环境

1. windows 下 Git Bash Here

配置全局用户名: git config –global user.name 你的用户名(一般是github用户名)

配置全局邮箱: git config –global user.email 你的邮箱(一般是github认证邮箱)

生成ssh密钥: ssh-keygen-t rsa -C 你的github认证邮箱

然后在 %userprofile%/.ssh目录下有三个文件,id_rsa,id_rsa.pub,known_hosts.

2. git基本使用

从github上clone一个项目到本地: git clone ssh/https

在本地初始化一个git仓库: git init

添加与本地仓库联系的远程仓库: git remote add 别名 ssh/https

别名为. 可以添加所有修改后的文件到缓存

修改文件后，将文件添加到本地仓库: git add .(files)

将修改后的内容上传到远程仓库: git push 别名 master(branch)

提交修改: git commit -m ‘你的commit’

这步必须,如果没会要求在git push 后输入(没提示)

提交过程应该连续,commit后再修改文件将push不成功

从远程仓库下载最新内容到本地仓库: git pull 别名 master(branch)

删除本地仓库与远程仓库的关联: git remote rm 别名

3. 其他命令

status命令

git status : 显示工作树的状态,一般有三种

  1. Untracked files: 未被跟踪的文件,表示是工作目录增加的文件
  2. Changes not staged for commit: 工作目录修改了文件,但是没有被添加到暂存区
  3. Changes to be committed: 添加到暂存区的文件,等待提价

reset命令

git reset : 将暂存区的所有文件重置到当前分支的HEAD

  1. git reset files : 将暂存区的指定文件重置到指定的
  2. git reset [–hard|soft|mixed|merge|keep] []
    mode的取值可以是hard,soft,mixed,merged,keep.
    hard:重置暂存区与工作目录到指定提交,删除之后的所有提交并将HEAD指向该提交,此操作危险指数较大(慎用)
    soft:暂存区与工作目录不会变化,仅仅删除之后的所有提交并将HEAD指向该提交
    mixed:默认的,重置暂存区到指定提交,删除之后的所有提交并将HEAD指向该提交
    merge与keep …
    …
